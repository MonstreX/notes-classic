ТЗ: Создание локального приложения "Notes Classic"
Стек: Tauri (Rust) + React (TypeScript) + Tailwind CSS + SQLite. Стиль: Строгое соответствие визуалу Evernote 2020 (как на скриншотах).

1. Визуальные требования (UI/UX)
Компоновка: Трехпанельный интерфейс (Navigation Menu | Note List | Note View).

Цветовая палитра: * Левая панель: Темно-антрацитовый (#1A1A1A). Акцентная кнопка "New Note": ярко-зеленый (#00A82D).

Средняя панель: Светло-серый фон, карточки заметок с белым фоном и тонкими разделителями.

Правая панель: Чисто белый фон для редактора.

Шрифты: Без засечек (Sans Serif), максимально близкие к системным (Segoe UI / San Francisco).

2. Функционал редактора (TipTap)
WYSIWYG: Базовое форматирование + поддержка таблиц и чек-листов.

Блоки кода: Интеграция расширения CodeBlockLowlight для TipTap с использованием библиотеки lowlight (на базе highlight.js).

Должна быть поддержка автоопределения языка или выпадающий список (JS, Python, C#, Rust, HTML/CSS).

Оформление блока кода: темная подложка, моноширинный шрифт.

Вставка (Paste): При вставке из браузера сохранять базовую HTML-структуру, очищая только скрипты.

3. База данных и локальность
SQLite: Хранить всё в одном файле notes_classic.db.

Таблица заметок: Добавить поле sync_status (INTEGER: 0 - локально, 1 - синхронизировано) и remote_id (TEXT) для будущего облака.

Хранение ресурсов: Картинки, вставленные в заметки, сохранять в локальную папку attachments/ и ссылаться на них в HTML, чтобы не раздувать БД.

4. Задел под синхронизацию (Future Proof)
Реализовать Repository Pattern в Rust-слое. Взаимодействие с данными должно идти через абстракцию, чтобы позже можно было добавить CloudSyncService без переписывания всего кода.

Предусмотреть в настройках заглушку (Placeholder) "Sync Settings".

Инструкция для ИИ-агента (что отправить в чат):
Используй проект Tauri + React + Vite.

Создай UI в точности как на прикрепленных изображениях: левая панель #1A1A1A, кнопка создания #00A82D.

Внедри TipTap с поддержкой расширения CodeBlockLowlight для подсветки синтаксиса кода.

Организуй хранение в SQLite. Каждая заметка — это запись с HTML-контентом.

Реализуй автоматическое сохранение (autosave) при изменении текста.

Добавь в схему БД поля для будущей синхронизации (sync_status, remote_id).
